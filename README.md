# 🎯 SalesCoach AI — Transcript Analyzer

> Paste a sales call transcript. Get instant AI-powered coaching signals — buying interest, objections, and confusion — with actionable one-line tips.

Built with **FastAPI** (Python backend) + **Google Gemini API** (LLM analysis) + **Tailwind CSS** (frontend).

---

## 📸 Preview

The app analyzes meeting transcripts and returns color-coded signal cards:

| Badge | Signal Type | Description |
|-------|-------------|-------------|
| 🟢 Green | **Buying Interest** | Prospect shows intent to purchase |
| 🔴 Red | **Objection** | Pushback on price, features, timeline, etc. |
| 🟡 Yellow | **Confusion** | Prospect is unclear about something |

Each card shows the **exact quote** from the transcript and a **coaching tip** generated by Gemini.

## 🚀 Getting Started

### Prerequisites

- Python **3.9+**
- A **Google Gemini API key** (free tier available) → [Get one here](https://aistudio.google.com/app/apikey)

---

### 1 — Clone the repository

```bash
git clone https://github.com/your-username/sales-coach-ai.git
cd sales-coach-ai
```

---

### 2 — Set up the Python virtual environment

```bash
# Create a virtual environment
python -m venv venv

# Activate it
# On macOS / Linux:
source venv/bin/activate

# On Windows (Command Prompt):
venv\Scripts\activate.bat

# On Windows (PowerShell):
venv\Scripts\Activate.ps1
```

Install dependencies:

```bash
pip install -r backend/requirements.txt
```

---

### 3 — Configure your Gemini API key

Copy the example environment file:

```bash
cp backend/.env.example backend/.env
```

Open `backend/.env` in any text editor and add your key:

```env
GEMINI_API_KEY=your_actual_gemini_api_key_here
```

> ⚠️ **Security:** Never commit `.env` to Git. It's already in `.gitignore`.

---

### 4 — Run the backend

```bash
cd backend
uvicorn main:app --reload
```

You should see:

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
```

You can verify it's working by visiting: [http://127.0.0.1:8000/health](http://127.0.0.1:8000/health)

---

### 5 — Open the frontend

No build step needed. Simply open the HTML file in your browser:

```bash
# macOS
open frontend/index.html

# Linux
xdg-open frontend/index.html

# Windows
start frontend/index.html
```

Or just drag-and-drop `frontend/index.html` into your browser.

> **Tip:** You can also use VS Code's [Live Server extension](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) for a better dev experience.

## 🧠 How It Works

1. You paste a sales call transcript into the textarea and click **Analyze Transcript** (or press `Ctrl+Enter`).
2. The frontend sends a `POST` request to `http://127.0.0.1:8000/analyse` with the transcript.
3. The FastAPI backend forwards it to **Google Gemini** (`gemini-2.0-flash`) with a structured system prompt.
4. Gemini returns a **pure JSON** list of signals — type, quote, and coaching tip.
5. The frontend renders them as animated, color-coded cards with a stats summary bar.

## 📡 API Reference

### `POST /analyse`

**Request body:**
```json
{
  "transcript": "Prospect: This pricing seems high...\nRep: Let me walk you through the ROI..."
}
```

**Response:**
```json
{
  "signals": [
    {
      "type": "objection",
      "quote": "This pricing seems high",
      "tip": "Acknowledge the concern, then pivot immediately to ROI and payback period."
    },
    {
      "type": "buying_interest",
      "quote": "We'd want to move quickly before Q4",
      "tip": "Lock in a specific date — ask 'Would the first week of October work to get contracts signed?'"
    }
  ]
}
```

Signal types: `"buying_interest"` | `"objection"` | `"confusion"`

---

### `GET /health`

Returns `{ "status": "ok" }` — useful for uptime checks.

## 🔒 Security Notes

- Your Gemini API key lives **only** in `backend/.env` (server-side) and is never exposed to the browser.
- The `.env` file is listed in `.gitignore` — never remove that entry.
- For production, restrict your API key in [Google AI Studio](https://aistudio.google.com/api-keys) to **Gemini API only** and optionally add IP restrictions.
- Enable CORS restrictions in `main.py` for production by replacing `allow_origins=["*"]` with your actual frontend domain.
